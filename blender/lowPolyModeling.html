Low Poly and Optimized Modeling
Video game optimization is something which is easy to complain about. And back in 2023 I had a ancient laptop, so everything ran slow. This got me thinking, how could I create the most optimized rendering possible?
Low Poly
So I got into low poly modeling. Which is modeling objects with the lowest number of triangles possible. And looking back this really is'nt the route to make video games render more quickly. As after doing some studying of graphics programming, there is a ton more going on behind the scenes. And much of the time the method for dealing with poly counts is having multiple LODs instead of one low poly mesh.
Even if low poly modeling did'nt allow me to create the most efficient models, it did allow me to create a lot of them. As skipping most of the refining I was able to mass produce these models.
Baking Lighting
So what are some other ways to create super optimized rendering? Well I tried precalculating a scenes lighting.

Baked lighting is a system where instead of calculating lighting real time, you do the math and save the result in a texture. Then those textures can be used to efficient re-render the scene.

The result I got was ok, you can see some issues at the borders of objects. And it appears I didn't use a Denoiser, maybe this is because I thought it looked cool. But if I want to improve this result in the future I would combine the baked lighting with real time lighting as well. Because baked lighting can't do reflections or update real time, so mixing other techniques in is necessary.
