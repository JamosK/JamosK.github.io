Modeling Trees
Early when learning how to 3D model, I got a obsession with trees. As unlike the engineered objects I was modeling, trees were incredibly organic and complicated. And I had seen fairly realistic trees in videos games.
So I was stuck thinking how? As every method I could imagine would require mass amounts of work manually placing curves
It turns out there is a very reasonable way how. Tree generation programs have been developed for this exact use. And blender has a plugin Sapling Tree Gen would is the exact tool for this task.
So I got to work, and modeled my first tree.

If you take a close took at that tree, it is'nt very tree like. It looks too much like plastic, the roots don't look right, and there are simply too few leaves.

After evaluating I identified some issues. I completely messing up the trees UV mapping. As the UV map shown on the right is a absolute mess. Being a simple one direction projection which is not sufficient for a cylinder.

I also should have used a leaf texture with multiple leaves. As to increase the density of leaves I should have used a texture which is essentially a mini branch.

And lastly the leaves where too random. As in real trees have a level of logic behind where leaves are place, and I need to somewhat replicate that logic using mathematics.
So I went and modeled my second tree.

This new tree still does'nt quite have the appearance of a tree. But I did fix many of the issues. I used a branch texture for the leaves, I fixing the UV mapping issues.

And I used geometry nodes for the leaves placement. So I could have more fine detail control over what angle every branch was placed at. Using that control I created a node tree which tries to replicate how trees angle their branches.
but issues persist. Such as despite my geometry nodes the leaves placement still does'nt look right. Specifically there are too many trees in the inners of the tree where light does'nt reach. Such as when looking at the shadow of the tree, it is a solid shadow. But the reference tree at the top of the page has many holes in the canopy.
Wanting to improve on leaf placement, I want and modeled it again.

This is the best tree yet but it still has errors. The branches you can now see, are far too sharp and straight for a real tree.

But biggest of all the leaf placement still is'nt quite right. As in many areas it is just far too dense for a real tree, and in other areas it is too sparse. So going forward I would probably need a completely distinct method of placing leaves, which models how real trees choose to grow leaves.
